The rapid growth of digital payment platforms such as PhonePe has significantly transformed financial transactions in India, making it essential to analyze user behavior, transaction patterns, and regional performance to improve services and strategic decision-making. This project focuses on analyzing aggregated transaction, user, insurance, and mapping data to derive meaningful insights that can support business growth, customer engagement, and operational efficiency. The primary objective is to understand how different regions, categories, and user groups contribute to overall platform activity and to identify key trends that can guide future improvements.

The project addresses multiple business use cases, including customer segmentation, fraud detection, geographical insights, payment performance evaluation, user engagement optimization, product development, insurance analysis, marketing optimization, trend analysis, and competitive benchmarking. By segmenting users based on transaction behavior, businesses can design personalized marketing strategies and improve customer retention. Transaction pattern analysis also helps in identifying anomalies that may indicate fraudulent activity, ensuring a secure payment ecosystem. Geographical analysis enables the identification of high-performing and underpenetrated regions, which supports targeted expansion and resource allocation.

To achieve these objectives, the data was first extracted from a GitHub repository containing structured PhonePe transaction datasets. The data was then loaded into a relational SQL database to ensure efficient storage and querying. Multiple tables were created to organize the data into aggregated, map, and top-level categories for users, transactions, and insurance information. Aggregated tables stored summarized user and transaction data, map tables captured state and district-level distribution of transaction values, and top tables highlighted the highest-performing users, regions, and categories.

Once the database was structured, SQL queries were used to extract relevant insights for analysis. The processed data was then analyzed using Python libraries such as Pandas for data manipulation and Plotly for visualization. Various charts including bar plots, pie charts, scatter plots, heatmaps, and line graphs were used to represent transaction trends, user engagement patterns, brand performance, and geographical distribution. These visualizations helped in identifying key patterns such as dominant states, high-performing brands, and variations in transaction types across regions.

A Streamlit-based interactive dashboard was developed to present the findings in a user-friendly manner, enabling real-time exploration of insights. This dashboard allows users to dynamically analyze transaction trends, compare regional performance, and understand category-wise contributions.

From the analysis, several important insights were derived. A small number of states and brands were found to contribute significantly to overall transaction volume, indicating concentrated usage patterns. Transaction trends showed consistent growth in user activity, although revenue growth was not always proportional, suggesting variations in transaction value behavior. Additionally, differences in regional adoption highlighted opportunities for expansion in underperforming areas.

Overall, the project demonstrates how data-driven analysis can be effectively used to understand complex digital payment ecosystems. The insights generated can support better decision-making in customer segmentation, marketing strategies, product development, and regional expansion. By leveraging these findings, businesses can enhance user engagement, optimize revenue generation, and ensure sustainable long-term growth in the evolving digital financial landscape.
